# baseline-sample/

This folder is the **curated** portfolio artefact: a known-good NBomber
report from a representative full 25-pair / 50-bot / 120s baseline run
against the local stack.

Reports generated by everyday `dotnet run -- load` invocations land
directly under `reports/` and are gitignored — only this `baseline-sample/`
subtree is committed.

## How a baseline was produced

```bash
docker compose -f docker-compose.yml \
  -f observability/docker-compose.observability.yml \
  -f observability/docker-compose.observability.override.yml \
  up -d

dotnet run --project tests/Kombats.LoadTests/SeedUsers -- --count 50

dotnet run --project tests/Kombats.LoadTests -- load
# default: --count 25 --duration 120
```

After the run finishes, NBomber drops a timestamped folder under
`reports/`. Copy the contents into `baseline-sample/<date>/` and commit.
Include the HTML, the CSV, and the markdown summary — they're all useful
to portfolio readers who can't or don't want to run the harness.

## What to look at first

- `index.html` — NBomber's per-step latency histograms. The key step is
  `battle_session` (one full bot lifecycle). p50/p95/p99 latency tell the
  story.
- `cluster-stats.txt` / `report.md` — text summary including success and
  failure rates.

## Grafana panels worth screenshotting alongside

- `active_battles{service="battle"}` over the run window — should rise to
  ~25 and fall back.
- `turn_resolution_duration_milliseconds` histograms.
- `db_client_connections_busy{db_system="postgresql"}` per service — first
  expected bottleneck (Postgres pool max=20 per service).
